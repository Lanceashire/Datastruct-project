图书管理系统

#### 一、问题描述

本实验针对**高校图书馆日常管理需求**，设计并实现一个控制台版图书管理系统。系统需满足**学生、教师、图书馆管理员**三类用户的不同操作权限，完成图书信息管理、借阅管理、借阅排行统计、图书分类管理等核心业务，并支持文件数据持久化。

系统主要业务包括：

1\.**图书入库管理**：图书信息添加、删除、修改、查询；

2\.**图书借阅与归还**：用户借阅、归还图书，自动更新库存；

3\.**借阅排行统计**：按借阅次数对图书进行排序展示；

4\.**图书分类管理**：支持图书分类添加、修改、查询；

5\.**文件管理**：支持文件创建、打开、保存、加载，保证数据持久化；

6\.**文本编辑**：支持文件内容的增、删、改、查、插入、替换；

7\.**权限分级**：不同用户拥有不同操作权限，防止越权操作；

8\.**扩展功能**：借阅临期提醒、超期提醒。

系统要求数据查询操作响应时间短，运行稳定，数据不丢失，界面清晰易用。

#### 二、系统分析

###### 1\.用户需求（从使用者的视角）

（1）系统用户类型

系统共划分三类角色，权限逐级区分，实现分级管控：

1\.系统管理员：图书馆工作人员，拥有系统最高权限，负责所有数据维护、功能配置；

2\.教师：校园读者，仅可查询图书、借阅 / 归还图书、查看个人借阅记录、接收借阅提醒；

3\.学生：校园读者，权限与教师一致，仅使用基础借阅、查询功能。

（2）业务需求

1\.**管理员业务需求**

- 图书业务：新增图书、删除废弃图书、修改图书信息、按条件查询图书、维护图书分类；

- 读者业务：登记新读者、注销读者账号、修改读者信息、查询所有读者信息；

- 借阅业务：办理借还手续、查询任意图书 / 读者的借阅记录、查看全局借阅排行；

- 文件业务：创建数据文件、加载历史数据、保存最新数据、编辑文件内容。

2\.**教师 / 学生业务需求**

- 图书查询：根据编号、书名、作者检索图书，查看图书库存与分类；

- 借阅操作：借阅可借图书、归还已借图书；

- 记录查询：查看个人所有借阅记录；

- 消息提醒：接收图书临期、超期提示。

（3）数据 IO 需求

·  **数据输入**：手动录入图书编号、书名、作者、分类、库存；读者编号、姓名、性别、联系方式；借阅编号、借阅时间、归还时间等。输入时增加合法性校验，禁止空值、非法字符、重复编号。

·  **数据输出**：以规整文本格式展示列表信息，查询结果逐条展示，每步操作反馈成功 / 失败提示。

·  **数据持久化**：所有图书、读者、借阅数据统一存储在本地\.txt文本文件，程序启动自动加载文件数据，程序退出自动将内存数据回写至文件。



（4）性能与运行环境要求

·  运行环境：Windows、Linux 全平台，纯控制台运行，无图形界面；

·  数据容量：支持 100 条以上图书、读者数据存储；

·  性能要求：增、删、改操作即时响应，查找、排序算法执行高效，文件读写稳定，无数据丢失、乱码问题。

7\.**系统退出模块** 功能：执行数据保存操作，安全关闭程序。 对应需求：正常终止程序，落地最新数据。

### （3）数据逻辑结构

三类数据均为**线性逻辑结构**，数据元素之间为一对一的前后顺序关系。整体采用**顺序表**作为逻辑载体，顺序表支持高效遍历、查找、排序，适配本系统高频查询、统计、遍历的业务场景。

#### （4）数据对象依赖关系

·  借阅记录依赖图书对象和读者对象：借阅记录中的bookId必须在图书列表中存在，readerId必须在读者列表中存在，编号无效则无法完成借阅操作，保证数据关联合法性。

·  图书库存与借阅记录联动：图书成功借阅时，对应图书库存数量自动减 1；图书完成归还时，对应图书库存数量自动加 1。

·  借阅统计数据联动：每新增一条有效借阅记录，对应图书的借阅次数自动加 1，为借阅排行统计提供数据支撑。

#### （5）数据对象基本操作及函数原型（抽象数据类型 ADT）

结合数据逻辑结构与对象依赖关系，分别定义图书、读者、借阅记录三类抽象数据类型，并说明对应基础操作、函数名称、参数及返回值。

##### ① 图书 ADT 操作

抽象数据类型图书线性表，数据元素为图书信息，元素之间呈线性顺序关系，配套操作如下：

1\.void AddBook \(BookList \*L, Book b\)：新增图书，参数为图书线性表指针、图书结构体变量，函数无返回值。

2\.int DelBook \(BookList \*L, int bookId\)：按图书编号删除图书，参数为图书线性表指针、目标图书编号；返回 1 代表删除成功，返回 0 代表删除失败。

3\.int ModBook \(BookList \*L, int bookId, Book newBook\)：修改指定编号的图书信息，参数为图书线性表指针、目标图书编号、新图书数据；返回 1 代表修改成功，返回 0 代表修改失败。

4\.Book \*SearchBook \(BookList \*L, int bookId\)：按编号查询图书，参数为图书线性表指针、目标图书编号；查询成功返回对应图书地址，未查询到则返回空地址。

5\.void ShowAllBook \(BookList \*L\)：遍历并展示所有图书信息，参数为图书线性表指针，函数无返回值。

6\.void UpdateStock \(BookList \*L, int bookId, int num\)：更新图书库存数量，用于配合借阅、归还业务实现数据联动；参数为图书线性表指针、图书编号、库存变动数值，函数无返回值。

7\.void UpdateBorrowCount \(BookList \*L, int bookId\)：图书借阅次数自增，用于统计借阅频次；参数为图书线性表指针、图书编号，函数无返回值。

##### ② 读者 ADT 操作

抽象数据类型读者线性表，数据元素为读者信息，元素之间呈线性顺序关系，配套操作如下：

1\.void AddReader \(ReaderList \*L, Reader r\)：新增读者，参数为读者线性表指针、读者结构体变量，函数无返回值。

2\.int DelReader \(ReaderList \*L, int readerId\)：按读者编号删除读者，参数为读者线性表指针、目标读者编号；返回 1 代表删除成功，返回 0 代表删除失败。

3\.int ModReader \(ReaderList \*L, int readerId, Reader newReader\)：修改指定编号的读者信息，参数为读者线性表指针、目标读者编号、新读者数据；返回 1 代表修改成功，返回 0 代表修改失败。

4\.Reader \*SearchReader \(ReaderList \*L, int readerId\)：按读者编号查询读者，参数为读者线性表指针、目标读者编号；查询成功返回对应读者地址，未查询到则返回空地址。

5\.void ShowAllReader \(ReaderList \*L\)：遍历并展示所有读者信息，参数为读者线性表指针，函数无返回值。

##### ③ 借阅记录 ADT 操作

抽象数据类型借阅记录线性表，数据元素为借阅信息，元素之间呈线性顺序关系；该数据对象依赖图书、读者对象，bookId、readerId必须为已存在有效编号，配套操作如下：

1\.void AddBorrow \(BorrowList \*L, Borrow br\)：新增借阅记录，同时联动更新图书库存与借阅次数；参数为借阅记录线性表指针、借阅记录结构体变量，函数无返回值。

2\.int ReturnBook \(BorrowList \*L, int borrowId, char \*retTime\)：办理图书归还业务，更新借阅记录并恢复对应图书库存；参数为借阅记录线性表指针、借阅编号、归还时间；返回 1 代表归还成功，返回 0 代表归还失败。

3\.void SearchBorrowByBook \(BorrowList \*L, int bookId\)：按图书编号查询该图书所有借阅记录；参数为借阅记录线性表指针、图书编号，函数无返回值。

4\.void SearchBorrowByReader \(BorrowList \*L, int readerId\)：按读者编号查询该读者所有借阅记录；参数为借阅记录线性表指针、读者编号，函数无返回值。

5\.void CheckRemind \(BorrowList \*L\)：遍历借阅记录，检测图书临期、超期状态并输出提醒信息；参数为借阅记录线性表指针，函数无返回值。

###### 4\.系统功能封装

定义各模块顶层封装函数，明确参数、返回值、功能及函数调用关系。

### （1）文件操作函数（数据持久化）

1\.int LoadData\(BookList \*bL, ReaderList \*rL, BorrowList \*brL\) 参数：图书、读者、借阅线性表指针；功能：从本地文件加载数据到内存；返回 1 = 加载成功，0 = 加载失败。

2\.int SaveData\(BookList \*bL, ReaderList \*rL, BorrowList \*brL\) 参数：三类线性表指针；功能：将内存数据写入本地文件；返回 1 = 保存成功，0 = 保存失败。

### （2）菜单与权限函数

1\.int Login\(\) 功能：用户登录，选择角色；返回值：1 = 管理员，2 = 教师，3 = 学生。

2\.void MainMenu\(int role, BookList \*bL, ReaderList \*rL, BorrowList \*brL\) 参数：用户角色、三类线性表指针；功能：根据角色加载对应主菜单，调度所有子功能。

### （3）业务模块入口函数

1\.void BookManage\(BookList \*bL\) 功能：图书管理子菜单入口，调度图书增删改查、分类、排序功能；无返回值。

2\.void ReaderManage\(ReaderList \*rL\) 功能：读者管理子菜单入口，调度读者增删改查功能；无返回值。

3\.void BorrowManage\(BookList \*bL, ReaderList \*rL, BorrowList \*brL, int role\) 参数：三类线性表、用户角色；功能：借阅业务子菜单，区分权限开放功能；无返回值。

4\.void FileEditMenu\(\) 功能：文件编辑子菜单，实现文件读写、文本编辑；仅管理员可用。

### （4）整体函数调用关系

程序入口 → LoadData\(\) 加载历史数据 → Login\(\) 身份校验 → MainMenu\(\) 主菜单

- 管理员菜单：调用BookManage、ReaderManage、BorrowManage、FileEditMenu

- 教师 / 学生菜单：仅调用BookManage\(查询\)、BorrowManage\(借还、查记录、提醒\) 选择退出 → SaveData\(\) 保存数据 → 程序结束

###### 2\.数据结构设计与实现

### （1）物理存储结构选择

###### 3\.算法设计

本节使用自然语言描述所有核心算法流程，不含代码。

### （1）顺序表基础操作算法

#### ① 新增元素（图书 / 读者 / 借阅记录）

7. 判断顺序表当前长度是否达到最大容量，已满则提示 “存储空间不足”，结束操作；

8. 遍历顺序表，校验编号是否重复，重复则提示 “编号已存在”，结束操作；

9. 将新数据存入顺序表末尾位置；

10. 顺序表长度 \+ 1，提示新增成功。

#### ② 按编号删除元素

11. 遍历顺序表，查找目标编号对应的元素下标；

12. 未找到则返回删除失败；

13. 从目标下标开始，将后续所有元素**向前平移一位**，覆盖被删除元素；

14. 顺序表长度 \- 1，返回删除成功。

#### ③ 按编号修改元素

15. 遍历顺序表定位目标编号；

16. 无匹配数据则返回失败；

17. 用新数据覆盖原结构体所有字段；

18. 返回修改成功。

#### ④ 按编号查询元素

19. 从头至尾遍历顺序表；

20. 比对编号，匹配成功则返回当前元素地址；

21. 遍历结束无匹配项，返回空地址。

#### ⑤ 全量遍历展示

22. 循环遍历顺序表中所有有效元素（下标 0 \~ 长度 \- 1）；

23. 按固定格式逐行输出每个字段信息。

### （2）核心业务算法

#### ① 图书借阅算法

24. 输入读者编号、图书编号，分别在读者表、图书表中校验编号合法性；

25. 校验图书库存，库存为 0 则提示 “暂无库存，无法借阅”；

26. 生成唯一借阅编号，录入借阅时间、归还截止时间，returnTime标记为 “未归还”；

27. 向借阅记录表新增一条记录；

28. 对应图书库存 \- 1、借阅次数 \+ 1；

29. 提示借阅成功。

#### ② 图书归还算法

30. 输入借阅编号，在借阅记录表中定位记录；

31. 校验状态：若已归还，提示 “该图书已完成归还”；

32. 录入当前日期作为归还时间，更新借阅记录；

33. 对应图书库存 \+ 1；

34. 提示归还成功。

#### ③ 借阅排行排序算法（冒泡排序）

需求：按图书borrowCount借阅次数**降序**排列。

35. 以图书顺序表为排序对象，设置外层循环控制排序轮数；

36. 内层循环依次比较相邻两个图书的借阅次数；

37. 若前一个元素借阅次数小于后一个，则交换两个元素位置；

38. 每一轮结束后，当前最大值沉到末尾；

39. 全部轮次执行完成后，遍历输出排序结果，即为借阅排行。

#### ④ 临期 / 超期提醒算法

40. 遍历所有未归还的借阅记录；

41. 取出当前系统日期、借阅截止日期进行比对： 

    1. 剩余天数≤3 天：标记为**临期**，输出提醒信息；

    2. 当前日期 \> 截止日期：标记为**超期**，输出提醒信息；

42. 遍历完成，汇总所有提醒内容。

### （3）文件读写算法

#### ① 加载数据算法

43. 以只读方式打开目标 txt 文件，文件不存在则创建空文件并直接返回；

44. 逐行读取文件内容，按逗号分割字符串，拆分出各个字段；

45. 将字段转换为对应数据类型，存入顺序表；

46. 全部读取完成，关闭文件。

#### ② 保存数据算法

47. 以写入模式清空原文件；

48. 遍历顺序表中所有有效元素，将每个字段按 “英文逗号” 拼接为一行字符串；

49. 逐行写入文件；

50. 写入完成，关闭文件。

#### 四、调试与测试

## 开发调试过程及问题解决

系统采用**模块化分步开发、逐模块调试**的方式实现，开发过程中遇到的典型问题及解决方案如下：

### 问题 1：文件读写乱码、字段拆分错误

- 现象：读取 txt 文件时，中文出现乱码，逗号分割字段错位，数据解析失败。

- 原因：文件编码不统一，换行符、多余空格干扰字符串分割。

- 解决：统一文件编码为 ANSI；读写前自动去除字符串首尾空格，严格按单个英文逗号分割字段。

### 问题 2：图书借阅后库存未更新

- 现象：办理借阅业务，借阅记录正常生成，但图书库存数值不变。

- 原因：函数参数传递为值传递，未修改原顺序表数据。

- 解决：所有操作函数统一使用**指针传递**，直接操作内存中原始顺序表。

### 问题 3：编号重复，导致数据冲突

- 现象：新增图书 / 读者时，手动输入重复编号，造成多条数据编号一致，查询逻辑混乱。

- 原因：未做编号唯一性校验。

- 解决：新增数据前，遍历整张顺序表，提前校验编号，重复则拒绝新增。

### 问题 4：临期 / 超期时间比对逻辑错误

- 现象：时间字符串比对结果不准确，提醒功能失效。

- 原因：直接用字符串比较日期，未转换为时间格式运算。

- 解决：将日期字符串拆分成年、月、日整型，通过数值运算计算天数差。

### 问题 5：顺序表越界

- 现象：删除数据后，顺序表长度未同步减 1，遍历访问到无效脏数据。

- 原因：删除元素后忘记修改length字段。

- 解决：所有增删操作严格同步更新顺序表有效长度。

###### 2．比较研究

本小组选用**顺序表 \+ 文本文件**方案，现对比其他可选存储方案，分析优劣：

### 方案 1：单链表存储（替代顺序表）

1\.实现思路：将顺序表替换为单链表，用链式结构存储图书、读者、借阅数据；

2\.优点：插入、删除元素无需移动数据，**频繁增删场景效率更高**，存储空间动态分配，无最大容量限制；

3\.缺点：不支持随机访问，按编号查找必须从头遍历，查询、排序效率低于顺序表；链表节点分散，文件持久化读写逻辑更复杂。

4\.适用场景：图书馆数据频繁新增、注销，查询操作较少的场景。

### 方案 2：数据库存储（替代文本文件）

1\.实现思路：使用 MySQL 等关系型数据库替代本地 txt 文件，数据存于数据库表中；

2\.优点：数据管理规范，支持复杂查询、多条件筛选、事务保护，**大数据量下读写、查询效率远高于文本文件**，天然支持数据持久化与数据校验；

3\.缺点：需要额外搭建数据库环境，增加开发复杂度，脱离本次 “纯数据结构 \+ 基础文件” 的课程实践范畴。

### 综合对比结论

结合本次课程实验定位（练习线性表、基础算法、文件操作），**顺序表 \+ 文本文件**是最优选择，结构简单、知识点贴合课程要求；若面向商用、大数据量图书馆系统，则优先选择**链表 \+ 数据库**组合。
